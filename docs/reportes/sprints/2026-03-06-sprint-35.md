## Task 35.1 — Response Compression Rollout ✅

**Goal:** Roll out Brotli + Gzip response compression to all remaining backend services.

### Before

- 4/25 services had compression (VehiclesSaleService, ComparisonService, VehicleIntelligenceService, Gateway)

### After

- **25/25 services** now have response compression (21 newly configured)

### Pattern Applied

```csharp
builder.Services.AddResponseCompression(options =>
{
    options.EnableForHttps = true;
    options.Providers.Add<BrotliCompressionProvider>();
    options.Providers.Add<GzipCompressionProvider>();
    options.MimeTypes = ResponseCompressionDefaults.MimeTypes.Concat(new[]
    {
        "application/json", "text/json", "application/problem+json"
    });
});
builder.Services.Configure<BrotliCompressionProviderOptions>(options => options.Level = CompressionLevel.Fastest);
builder.Services.Configure<GzipCompressionProviderOptions>(options => options.Level = CompressionLevel.Fastest);
```

## Task 35.3 — NotificationService Security Fixes ✅

**Goal:** Fix 11 deferred security findings from Sprint 32 audit.

### S-13 (Critical) — Webhook Fail-Closed ✅

- **Before:** Both SendGrid and Twilio webhooks were **fail-open** — when verification keys weren't configured, webhooks processed anyway
- **After:** Both return `503 Service Unavailable` when keys aren't configured (fail-closed)
- Files modified: `WebhooksController.cs`

### S-12 (High) — Template XSS Prevention ✅

- **Before:** `TemplateService.ResolveValue()` returned raw `current?.ToString()` — no HTML encoding
- **After:** Returns `WebUtility.HtmlEncode(current?.ToString())` — prevents stored XSS via template parameters
- Files modified: `TemplateService.cs`

### S-06 (Critical) — FluentValidation on Non-MediatR Endpoints ✅

- **Before:** 6 of 10 controllers bypassed MediatR, meaning ~25+ string fields had zero validation
- **After:** 8 new validators covering all non-MediatR request DTOs

| Validator                                      | String Fields Protected                                       |
| ---------------------------------------------- | ------------------------------------------------------------- |
| `CreateTemplateRequestValidator`               | Name, Subject, Body, Description, Category, Tags, PreviewData |
| `UpdateTemplateRequestValidator`               | Subject, Body, Description, Tags, PreviewData                 |
| `PreviewTemplateRequestValidator`              | TemplateId, TemplateContent                                   |
| `ScheduleNotificationRequestValidator`         | TimeZone, CronExpression (+ regex validation)                 |
| `UpdateNotificationPreferenceRequestValidator` | Type, Channels (RuleForEach)                                  |
| `CreateUserNotificationRequestValidator`       | UserId, Type, Title, Message, Icon, Link, DealerId, ExpiresAt |
| `BulkNotificationRequestValidator`             | UserIds (RuleForEach), Type, Title, Message, Icon, Link       |
| `TeamsAlertRequestValidator`                   | Title, Message, Severity (+ enum whitelist)                   |

- Created `FluentValidationFilter` (same pattern as ReportsService)
- Moved inline `TeamsAlertRequest` record from controller to `Application/DTOs/`

### S-04 (Medium) — Internal API Key Authentication ✅

- **Before:** `/api/internal/*` endpoints relied solely on K8s network boundary (AllowAnonymous)
- **After:** Created `InternalApiKeyMiddleware` that validates `X-Internal-Api-Key` header
- Fail-closed: if `Security:InternalApiKey` config isn't set, returns 503
- Registered in pipeline after auth middleware
